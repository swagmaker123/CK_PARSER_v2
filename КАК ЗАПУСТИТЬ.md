# Как запустить CK PARSER

Краткая шпаргалка. Подробнее — в [README.md](README.md).

Env (модели, ключи API): сначала `.env` в корне проекта, если нет — `~/.openclaw/.env`.

## Установка (один раз)

**Windows (PowerShell):**

```powershell
cd C:\path\to\CK_PARSER
python -m venv venv
.\venv\Scripts\activate
pip install -r requirements.txt
playwright install chromium
```

**Linux / macOS:**

```bash
cd /path/to/CK_PARSER
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
# Chromium + системные зависимости Linux (нужен sudo / apt)
playwright install --with-deps chromium
```

На Linux: `source venv/bin/activate` (не `.\venv\Scripts\activate`).

Если `sudo playwright ...` не видит команду из venv:

```bash
sudo ./venv/bin/playwright install-deps chromium
playwright install chromium
```

## Обычный прогон (шаги отдельно)

```bash
# 1) парсинг + фильтры → активный Excel
python main.py

# за один день
python main.py --days 1

# 2) суммаризация + score + dedupe
python main.py --enrich-only

# 3) топ-10 + 5 резерв (обычно раз в месяц; в .env: LLM_MAX_TOKENS=4500)
python main.py --rank-only --top-n 10 --reserve-n 5

# 4) почта (после send активный период → output/sent/)
python main.py --send-only output/news_FROM_TO.xlsx --plain
```

Можно совместить 1 и 2: `python main.py --enrich`.

## Полный пересбор за 30 дней

```powershell
python main.py --days 30 --refresh-days 30
```

## Только Excel из кэша (без сайтов)

```powershell
python main.py --export-only --days 30
```

Если меняли фильтры — сначала `python main.py`, потом `--export-only`.

## Один источник / один ЦК

```powershell
python main.py --source interfax
python main.py --ck taxes
```

## Где результат

Активный Excel на период (дописывается каждый день):

```text
output\news_YYYY-MM-DD_YYYY-MM-DD.xlsx
output\.active_workbook
```

После `--send` / `--send-only` по активному файлу период закрывается:

```text
output\sent\news_YYYY-MM-DD_YYYY-MM-DD.xlsx
```

Следующий прогон создаёт новый активный файл.

## Кэш и лог

```text
cache\{источник}\raw\cache.json       — сырой кэш
cache\{источник}\{ck_id}\cache.json   — после фильтра
logs\run_YYYY-MM-DD_HH-MM-SS.log     — лог прогона
```
