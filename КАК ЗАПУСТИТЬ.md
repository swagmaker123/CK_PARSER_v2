# Как запустить CK PARSER

Краткая шпаргалка. Подробнее — в [README.md](README.md).

## Установка (один раз)

**Windows (PowerShell):**

```powershell
cd C:\path\to\CK_PARSER
python -m venv venv
.\venv\Scripts\activate
pip install -r requirements.txt
playwright install chromium
```

**Linux / macOS:**

```bash
cd /path/to/CK_PARSER
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
# Chromium + системные зависимости Linux (нужен sudo / apt)
playwright install --with-deps chromium
```

На Linux: `source venv/bin/activate` (не `.\venv\Scripts\activate`).

Если `sudo playwright ...` не видит команду из venv:

```bash
sudo ./venv/bin/playwright install-deps chromium
playwright install chromium
```

## Обычный прогон

**Windows:** `.\venv\Scripts\activate`  
**Linux / macOS:** `source venv/bin/activate`

```bash
python main.py
```

Все источники, все ЦК (ПС, ССЭМ, Налоги), 30 дней → **только Excel**.

LLM и почта **не запускаются**. Для полного цикла:

```powershell
python main.py --enrich --send
```

(нужны ключ LLM, SMTP и файл `assets/email_header.png`)

## Полный пересбор за 30 дней

```powershell
python main.py --days 30 --refresh-days 30
```

## Только Excel из кэша (без сайтов)

```powershell
python main.py --export-only --days 30
```

Если меняли фильтры — сначала `python main.py`, потом `--export-only`.

## Один источник / один ЦК

```powershell
python main.py --source interfax
python main.py --ck taxes
```

## Где результат

```text
output\news_2026-06-23.xlsx
```

Один файл: все источники и все ЦК, колонки `Источник` и `Наименование ЦК` (ПС / ССЭМ / Налоги).

## Кэш и лог

```text
cache\{источник}\raw\cache.json       — сырой кэш
cache\{источник}\{ck_id}\cache.json   — после фильтра
logs\run_YYYY-MM-DD_HH-MM-SS.log     — лог прогона
```
