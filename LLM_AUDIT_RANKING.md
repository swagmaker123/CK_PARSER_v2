# LLM Audit Ranking

Этот режим делает LLM-постобработку готового Excel-файла с новостями.

Цель: Отобрать топ новостей по каждому ЦК с точки зрения аудиторской релевантности.

## Что делает

LLM audit ranking работает в три этапа:

1. Первый проход оценивает каждую новость отдельно.
   - Формирует нормальное краткое резюме новости для рассылки.
   - Ставит оценку релевантности для своего ЦК от `0` до `100`.
   - Решает, стоит ли передавать новость во второй этап.
   - Во второй этап проходят только новости с `llm_score >= 60`.

2. Semantic dedupe (опционально, между 1-м и 2-м проходом).
   - Берёт **все строки** каждого ЦК после первого прохода.
   - Embeddings через Cloud.ru (`EMBEDDING_MODEL` + `FOUNDATION_MODELS_API_KEY`).
   - Семантические дубли **удаляются из Excel** как строки.
   - В группе дублей остаётся новость с **максимальным `llm_score`**.
   - Если не заданы `EMBEDDING_MODEL` или API-ключ — этап пропускается.
   - Подробнее: [DEDUPE.md](DEDUPE.md).

3. Второй проход берёт только оставшихся кандидатов по каждому ЦК.
   - Если кандидатов больше `30`, во второй проход отправляются только 30
     новостей с максимальным `llm_score`.
   - Выбирает ранжированный список `top_n + reserve_n`.
   - По умолчанию это `10` основных новостей и `5` резервных.
   - Помечает новости, которые входят в итоговый топ-10.


Практический смысл порога:

- `90-100` — почти точно кандидат в итоговый топ.
- `75-89` — сильная новость для аудиторского мониторинга.
- `60-74` — пограничный кандидат, попадёт во второй этап.
- `55-69` — умеренно полезно, остаётся в Excel с оценкой и summary, но во второй этап не передаётся.
- `<55` — слабая или нерелевантная новость.

Промпты разные для каждого ЦК и лежат в:

```text
llm/ranking_prompts/
```

Сейчас поддерживаются:

- `payment_systems` — ПС
- `ssem` — ССЭМ
- `taxes` — Налоги



## Какие колонки появляются в Excel

Все старые колонки остаются:

- `Дата новости`
- `Заголовок статьи`
- `Полный текст`
- `Ссылка на источник`
- `Источник`
- `Наименование ЦК`
- `Ключевые слова`

Добавляются или обновляются:

- `llm_summary` — краткое резюме новости для рассылки, обычно 2-4 предложения
- `llm_score` — оценка релевантности новости для своего ЦК
- `top_rank` — место в итоговом ранжировании по своему ЦК, обычно от `1` до `15`
- `is_top_news` — `True`, если новость входит в топ-10 по своему ЦК

Служебные колонки `audit_*` и `_audit_*` в итоговый Excel не сохраняются.

## Как запустить после обычного парсинга

```powershell
python main.py --enrich
```

Команда сначала соберёт Excel, потом запустит LLM audit ranking.

## Как запустить по готовому Excel за сегодня

```powershell
python main.py --enrich-only
```

Если `--output` не указан, команда ищет файл:

```text
output/news_YYYY-MM-DD.xlsx
```

То есть за сегодняшнюю дату.

## Как запустить по конкретному файлу

```powershell
python main.py --enrich-only --output output/news_2026-06-25.xlsx
```

## Как запустить по конкретному ЦК

ПС:

```powershell
python main.py --enrich-only --ck payment_systems
```

ССЭМ:

```powershell
python main.py --enrich-only --ck ssem
```

Налоги:

```powershell
python main.py --enrich-only --ck taxes
```

С конкретным файлом:

```powershell
python main.py --enrich-only --output output/news_2026-06-25.xlsx --ck payment_systems
```

## Как изменить размер топа

По умолчанию:

- `--top-n 10`
- `--reserve-n 5`

То есть LLM выбирает 15 новостей: первые 10 основные, ещё 5 резерв.

Пример:

```powershell
python main.py --enrich-only --ck payment_systems --top-n 10 --reserve-n 5
```

Если нужен только топ-10 без резерва:

```powershell
python main.py --enrich-only --ck payment_systems --reserve-n 0
```

## Что нужно для работы

Нужен ключ LLM API:

```text
FOUNDATION_MODELS_API_KEY
```

`main.py` загружает `.env` из:

```text
~/.openclaw/.env
```

Пример переменных:

```text
FOUNDATION_MODELS_API_KEY=...
LLM_MODEL=ai-sage/GigaChat3-10B-A1.8B
LLM_TEMPERATURE=0.2
LLM_MAX_TOKENS=4096
LLM_TEXT_TRUNCATE=8000
```

Опционально для semantic dedupe (см. [DEDUPE.md](DEDUPE.md)):

```text
EMBEDDING_MODEL=BAAI/bge-m3
DEDUPE_SIMILARITY_THRESHOLD=0.88
```

## Важное замечание

В LLM уходит текст новости из колонки `Полный текст`.

Текст обрезается по лимиту:

```text
LLM_TEXT_TRUNCATE
```

По умолчанию используется `8000` символов.
